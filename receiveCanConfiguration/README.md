# receiveCanConfiguration

STM32F303K8で、CAN経由のモーター指令・PID設定・LED制御と、モーターおよびリミットスイッチの状態送信を行うファームウェアです。

## CAN基本仕様

- Classic CAN、標準11 bit ID、データフレームを使用します。
- 現在のクロック設定における通信速度は1 Mbpsです。
- STM32側の受信フィルタは`0x100`と`0x200`から`0x2FF`を通し、登録済みIDだけをソフトウェアコールバックで処理します。
- 複数バイト整数と`float32`はlittle-endianです。
- メイン制御周期は10 msです。

## CAN ID一覧

| CAN ID | 方向 | DLC | 内容 |
| --- | --- | ---: | --- |
| `0x100` | 外部 -> STM32 | 1以上 | heartbeat |
| `0x101` | STM32 -> 外部 | 1 | heartbeat応答 |
| `0x210` | 外部 -> STM32 | 4以上 | LED 1個の色を設定 |
| `0x230` | 外部 -> STM32 | 2以上 | motor1の目標RPM |
| `0x231` | 外部 -> STM32 | 2以上 | motor2の目標RPM |
| `0x232` | 外部 -> STM32 | 1以上 | motor3の出力率 |
| `0x240` | 外部 -> STM32 | 5以上 | motor1のPIDゲイン設定 |
| `0x241` | 外部 -> STM32 | 5以上 | motor2のPIDゲイン設定 |
| `0x320` | STM32 -> 外部 | 1 | リミットスイッチ状態 |
| `0x330` | STM32 -> 外部 | 2 | motor1の現在RPM |
| `0x331` | STM32 -> 外部 | 2 | motor2の現在RPM |
| `0x332` | STM32 -> 外部 | 2 | motor3の現在PWM |
| `0x340` | STM32 -> 外部 | 2 | motor1のPID設定結果 |
| `0x341` | STM32 -> 外部 | 2 | motor2のPID設定結果 |

`0x220`、`0x310`は使用しません。

## Heartbeat

外部制御器は`0x100`を定期送信します。データは1バイトで、カウンターなど任意の値を使えます。

```text
ID:     0x100
Byte 0: heartbeat値
```

STM32は受信した最新値を`0x101`から返します。CAN送信は受信IRQ内では行わず、次のメインループで実行します。送信メールボックスが使用中の場合は次回再試行します。

```text
ID:     0x101
Byte 0: 0x100で受信した最新値
```

heartbeatを1度も受信していない場合、または最終受信から500 msを超えた場合は安全停止状態になります。通常は100 ms以下の周期で送信することを推奨します。

安全停止中は次の動作になります。

- motor1、motor2、motor3のPWMを1000にします。
- 保存されていたモーター指令を破棄します。
- 安全停止中に受信した`0x230`、`0x231`、`0x232`は無視します。
- heartbeat復帰後は、新しいモーター指令の受信が必要です。

## モーター指令

### motor1 / motor2 RPM

`0x230`がmotor1、`0x231`がmotor2です。

```text
Byte 0: RPMの下位8 bit
Byte 1: RPMの上位8 bit
```

値は符号付き16 bit整数です。motor1とmotor2はエンコーダーから算出した現在RPMを使ってPID制御します。現在の制御では目標RPMが0以下の場合、PWMを停止値の1000にするため、逆回転指令には対応していません。

### motor3 出力率

`0x232`で0から100の出力率を指定します。

```text
Byte 0: 出力率 [%]
```

PWMへの変換式は次のとおりです。

```text
PWM = 1000 + 出力率 * 10
```

100を超える値は100に制限されます。したがって、0%はPWM 1000、50%は1500、100%は2000です。

## PID設定

`0x240`がmotor1、`0x241`がmotor2です。1フレームで1個のゲインを設定します。

```text
Byte 0: パラメータ番号
        0 = Kp
        1 = Ki
        2 = Kd
Byte 1-4: IEEE 754 float32、little-endian
```

例として、motor1の`Kp`を`0.05f`に設定するフレームは次のとおりです。

```text
ID:   0x240
DLC:  5
Data: 00 CD CC 4C 3D
```

設定結果はmotor1なら`0x340`、motor2なら`0x341`で返します。

```text
Byte 0: 受信したパラメータ番号
Byte 1: 結果
        0 = 失敗
        1 = 成功
```

DLC不足、不正なパラメータ番号、NaNまたはInfは失敗です。DLCが0でパラメータ番号を取得できなかった場合、応答のByte 0は`0xFF`になります。

PID設定コールバックは受信IRQ内でゲインを更新し、結果を応答キューへ追加します。CAN送信はIRQ内では行わず、次回以降の`motor_loop()`から行います。応答キューは最大15件を保持します。

結果の「成功」はPID値を受理して反映したことを示します。CANフレームが相手へ到達したことを保証する送達確認ではありません。

## モーターフィードバック

すべて2バイトlittle-endianです。

- `0x330`: motor1の現在RPM。符号付き16 bit整数。
- `0x331`: motor2の現在RPM。符号付き16 bit整数。
- `0x332`: motor3に現在設定しているPWM。符号なし16 bit整数。

10 msの制御ループごとに1フレームを`0x330`、`0x331`、`0x332`の順で送るため、通常は各IDが約30 ms周期になります。PID結果応答が保留中の場合はPID応答を優先するため、モーターフィードバック周期は延びます。

CANメールボックスへ追加できなかった場合、同じフィードバックIDを次回ループで再試行します。

## LED制御

`0x210`でLED 1個の色を設定します。

```text
Byte 0: LED番号、0から27
Byte 1: R、0から255
Byte 2: G、0から255
Byte 3: B、0から255
```

例として、LED 5を赤に設定するフレームは次のとおりです。

```text
ID:   0x210
DLC:  4
Data: 05 FF 00 00
```

範囲外のLED番号は無視されます。受信コールバックはLEDコマンドキューに追加するだけです。LEDバッファの更新とSK6812へのDMA転送開始はメインループから行います。LED状態のCAN返信はありません。

## リミットスイッチ

4個のリミットスイッチ状態を`0x320`で約10 msごとに送信します。外部からの要求フレームは不要です。

```text
Byte 0 bit 0: LIMIT_SW1
Byte 0 bit 1: LIMIT_SW2
Byte 0 bit 2: LIMIT_SW3
Byte 0 bit 3: LIMIT_SW4
```

各ビットはスイッチが押されてGPIOがRESETになったとき1です。CANメールボックスへ追加できなかった場合、その周期のフレームは破棄され、次の周期に最新状態を送ります。

## モーター構成

| 対象 | 制御 | PWM出力 | フィードバック |
| --- | --- | --- | --- |
| motor1 | RPM PID | TIM15 CH1 | TIM1 Encoder |
| motor2 | RPM PID | TIM3 CH1 | TIM2 Encoder |
| motor3 | 出力率から直接PWM | TIM3 CH2 | なし |
| LED | SK6812 DMA | TIM17 CH1 | なし |

PID初期値はmotor1、motor2ともに`Kp=0.05f`、`Ki=0.0f`、`Kd=0.0f`です。エンコーダー分解能は2048 PPR、PID計算周期は10 msとしてRPMを算出します。

## 実装上の注意

- CAN受信IRQではFIFOから受信後、CAN IDに登録された軽量コールバックを呼び出します。
- PID計算、PWM CCR更新、CAN送信、LED DMA開始はメインループで実行します。
- CANの自動再送とBus-Off自動復帰は有効です。
- heartbeat未受信または500 msタイムアウト時はモーターを安全停止します。
- PID結果とCAN送信処理は、メールボックスへの登録成功を基準に進行し、送信完了割り込みによる到達確認は行っていません。

## ビルド

STM32CubeIDEのDebug構成、または生成済みMakefileからビルドできます。

```sh
cd receiveCanConfiguration/Debug
make all
```
